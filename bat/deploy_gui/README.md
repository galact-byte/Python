# 通用部署器

基于 `PyQt6 + Paramiko` 的桌面部署工具，支持多项目配置、执行计划预览和多种部署模式，适合 ZIP 上传覆盖、服务器 `git pull`、自定义远端命令等常见维护场景。

## 当前特性

- 多项目配置保存与切换
- 三种部署模式：
  - ZIP 上传部署
  - Git 拉取部署
  - 自定义命令部署
- 执行前先生成计划预览
- SSH 测试连接
- 实时日志输出
- 密码每次手动输入，不落盘
- 默认浅色卡片式界面，日志区单独深色终端风格
- 左侧项目操作区改成紧凑工具头，新增/复制/删除和 `Program` 预设集中放置
- 左侧项目列表会显示每个项目的模式和目标主机，不再只是纯名称列表
- 首次打开会自动带一个 `Program` ZIP 部署预设
- 第一版仅支持 Linux 远端主机

## 安装

```powershell
cd E:\Python\Programs\test\bat\deploy_gui
python -m pip install -r requirements.txt
```

## 启动

```powershell
cd E:\Python\Programs\test\bat\deploy_gui
python main.py
```

## 使用流程

1. 新增或选择一个项目配置
2. 填写服务器、路径、部署模式和部署选项
3. 点击“保存配置”
4. 点击“生成计划”确认将执行的步骤
5. 点击“测试连接”确认 SSH 可用
6. 点击“开始部署”，输入密码后执行
7. 在右侧日志窗口查看执行过程和失败点

## 界面说明

- 左侧是项目导航和高频操作区
- 左侧可一键插入 `Program` 预设，适配你当前这套 ZIP 上传覆盖流程
- 左侧项目列表的每一项都会展示项目名、模式标签和主机摘要，列表本身承担导航主视图
- 中间是部署配置工作台，按基础配置、模式配置、部署选项、执行操作分组
- 右侧是执行计划与远端日志，日志区单独深色显示，便于区分运行态信息

## 部署模式说明

### ZIP 上传部署

适用于本地打包 ZIP，上传后在服务器上解压覆盖并执行构建/重启命令的场景。

典型步骤：
- 本地打包 ZIP
- 上传 ZIP
- 远端解压覆盖
- 安装依赖 / 前端构建 / 服务重启 / 健康检查

### Git 拉取部署

适用于服务器上已经存在 Git 仓库，直接 `git pull` 后继续执行部署命令的场景。

### 自定义命令部署

适用于已有固定命令流，但不适合 ZIP 或 Git 模式的场景。

## 配置保存位置

项目配置默认保存在：

```text
%USERPROFILE%\.deploy_gui\projects.json
```

密码不会保存到配置文件中。

## 当前限制

- 仅支持 Linux 远端主机
- 第一版不支持拖拽式流程编排
- 目前未做完整 GUI 自动化测试，主要依赖核心逻辑测试、窗口结构测试和手工验证
