# 视频 BGM 配乐合成工具

把一个**短视频**里的 BGM（背景音乐音轨）干净地循环铺到一个**长视频**上，
同时保留长视频的人声（音乐为主、人声清晰），输出新视频，**原文件不动**。

> 典型场景：长视频 = 短视频基础上增加了些内容（更长、有人声没 BGM），
> 短视频 = 带高质量 BGM（人声少/无）。想给长视频配上同样的音乐。

---

## 快速开始

**方式一：双击 `start.bat`**（推荐）

按菜单操作：
- `[1] 快速合成` —— 全自动，全程铺 BGM，循环无缝
- `[2] 自定义合成` —— 可设留白时段 / 音量 / 锚点
- `[3] 仅分析 BGM 结构` —— 看看音源的歌曲起点、强拍点、静音边界

路径可直接把视频文件拖进窗口（自动带引号也能识别）。

> **通用性**：每对视频都会**独立重新分析**（歌曲起点、强拍点、结尾静音都是现场探测）。
> 换任意歌曲 / 长度 / 内容都能直接用。`[1] 快速合成` 只问两个路径，其余全自动。
> 六花那套留白/跳间奏/锚点参数是**可选进阶功能**，默认不启用，普通需求用不到。
> 最简配置见 `config_最简示例.json`；进阶用法见 `config_六花示例.json`。

**方式二：用配置文件（适合批量/复用同一套参数）**

```
python bgm_tool.py --config config_六花示例.json
```

测试时只想先听音频、不想等视频合成：

```
python bgm_tool.py --config xxx.json --audio-only
```

## 依赖

- Python 3.7+
- ffmpeg / ffprobe（需在 PATH，或放在 `D:\Software\ffmpeg\bin`）
  - Windows 下载：https://www.gyan.dev/ffmpeg/builds/

无需任何 pip 第三方库。

---

## 核心技术（为什么接缝干净、音质好）

1. **音源取短视频原生音轨**（通常比单独下载的 mp3 码率高），最终编码 256k，不重采样。
2. **循环接缝只落在歌曲"真静音"处**（段落边界 / 歌曲结尾大静默），
   绝不在人声句子中间 crossfade —— 否则会出现两段人声重叠的违和感。
3. **软前奏只在开头播一次**，后续循环从"强拍"进入，避免循环回到软弱前奏。
4. **可选跳过安静间奏**：在静音里干净跳过 bridge，让新增内容也压得住音乐。
5. **留白区**（开场/中段对白）不放 BGM，只留人声。
6. **人声闪避**：人声出现时音乐轻微下压，人声停止恢复。
7. **两步渲染**：先合成小音频，再 copy 复用视频流 —— 快且抗 IO 卡顿。

## 配置字段说明

见 `config_六花示例.json`。关键字段：

| 字段 | 含义 | 默认 |
|:--|:--|:--|
| `short_video` / `long_video` | 短(带BGM) / 长(带人声) 视频路径 | 必填 |
| `output_dir` | 输出目录 | 长视频旁 `配乐合成/` |
| `bgm_volume` / `voice_volume` | BGM / 人声 相对音量 | 0.85 / 0.6 |
| `duck_ratio` | 人声出现时音乐下压强度（越大越狠） | 1.8 |
| `intro_once` | 软前奏只播一次，循环从强拍进 | true |
| `song_start` | 短视频里音乐真正起点(秒)，null=自动 | 自动 |
| `song_end` | 歌曲天然结尾/最长静音(循环边界)，null=自动 | 自动 |
| `loop_start` | 循环重新进入的强拍点(秒)，null=自动 | 自动 |
| `skip_ranges` | 要跳过的安静间奏 `[[起,止]]`，须落在静音里 | `[]` |
| `silence_zones` | 长视频里不放 BGM 的留白 `[[起,止]]` | `[]` |
| `anchors` | 让某段 BGM 从歌曲指定位置起播 | `[]` |

时间支持 `秒` 或 `分:秒`（如 `4:54`）两种写法（交互模式下）。

---

## 调参建议

- **音乐太小听不清** → 调大 `bgm_volume`、调小 `voice_volume`，或减小 `duck_ratio`。
- **某段没音乐**（被安静间奏盖住）→ 用 `[3]` 看内部静音点，把间奏填进 `skip_ranges`
  （**两端必须是静音点**，否则会出现人声重叠）。
- **循环处突兀** → 检查 `loop_start` 是否落在强拍上（`[3]` 能量采样里该点应明显更响）。
- **某段想从歌曲特定位置开始**（听习惯了）→ 用 `anchors`。

---

## 注意

- 原始视频文件不会被修改。
- 视频流是 `copy`（不重编码），所以画质无损、合成很快。
- 大视频合成时建议先关掉网盘客户端，避免占用磁盘 IO 导致写入变慢。
