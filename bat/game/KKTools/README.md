# KKTools — 恋活 / 恋活日光浴 角色卡工具箱

一个用 Python + PyQt6 写的桌面工具，用来管理 **恋活 (Koikatu / KK)** 与
**恋活日光浴 (Koikatu Sunshine / KKS)** 的角色卡：浏览、编辑、解包打包、伪装隐写、
Mod 缺失检测、场景卡提取、分享整理。

> 界面走克制的功能型暗色风格，不堆砌花哨装饰；所有耗时操作放后台线程，不卡界面。

---

## 功能模块

| 模块 | 说明 |
| :--- | :--- |
| **角色卡编辑** | 读写角色卡：姓名 / 昵称 / 性格 / 血型 / 社团 / 生日，以及 **提问 / 特点 / H相关** 三类勾选项与数值项；报告 / 卡片详情 / 解析JSON / 捏脸摘要四视图；更换预览图、导出 JSON、KK↔KKS 标识转换（实验性）。**默认另存为新卡，覆盖前自动备份。** |
| **卡片浏览器** | 扫描目录，识别角色卡 / 服装卡 / 场景卡，缩略图墙预览；类型筛选、名称搜索、双击进编辑器、导出卡片信息 CSV。 |
| **解包 / 打包** | zip 解包；把文件夹压缩后**追加到真实视频/图片之后伪装**（既能播放又能解压）；可分卷。解包能识别伪装在载体后的压缩包。 |
| **Mod 仓库** | 扫描本地 `.zipmod` 读取 `manifest.xml` 建立 GUID 索引（可缓存）；从角色卡的 Sideloader 扩展数据提取依赖，**比对找出缺失的 mod**，导出缺失清单。 |
| **场景卡工具** | 从 Studio 场景卡**提取内嵌角色卡**（以脸图作封面，重建为可独立读取的标准卡）；分析场景内角色与聚合的 mod 依赖。 |
| **分享整理导入** | 把角色卡与其依赖的 mod **一起打包成分享包**（依赖取自 Mod 索引），缺失项记录在 README。 |
| **运行日志 / 软件设置** | 操作记录；配置游戏目录、Mod 目录、输出目录等。 |

## 安装与运行

### 方式一：双击启动（Windows）

直接双击 **`start.bat`**。首次运行会自动检查 Python 与依赖并补齐，然后启动程序。

### 方式二：命令行

```bash
python launch.py        # 带依赖自检的启动器
# 或
pip install -r requirements.txt
python main.py
```

**环境要求**：Python 3.9+，依赖 `PyQt6` / `msgpack` / `Pillow`。

## 快速上手

- **编辑一张卡**：把角色卡 PNG 拖进窗口，或在「角色卡编辑」点「读卡」。改完用「另存为新卡」。
- **整理收藏**：「卡片浏览器」选目录扫描，按类型筛选，导出 CSV 管理。
- **查缺 mod**：「软件设置」填好游戏目录 →「Mod 仓库」建立索引 → 拖卡进检查队列。
- **从场景扒人**：「场景卡工具」选场景卡 →「提取角色到目录」。
- **打包分享**：「分享整理导入」加卡 + 选输出目录 → 生成分享包（含依赖 mod）。

## 技术说明

恋活角色卡是 **PNG 图片 + 尾部追加的二进制块**（MessagePack 序列化），核心解析器
`core/kk_card.py` 实现了安全的读写：

- **未编辑时字节级原样写回**——读进来再存出去与原文件完全一致；
- 编辑时只重打包被改动的块（如 Parameter），其余块（Custom / Coordinate / KKEx /
  Status / About）**字节级保留**，最大限度避免破坏卡片；
- 已在真实 KK / KKS 角色卡上验证往返一致性与编辑安全性。

项目结构：

```
core/        与界面无关的核心逻辑（卡片解析、Mod 索引、隐写、场景、分享、配置）
ui/          PyQt6 界面（主窗、各功能页、样式、后台线程、日志）
tests/       核心解析器的单元测试
launch.py    启动器（依赖自检）
main.py      程序入口
```

运行测试：

```bash
python tests/test_kk_card.py
```
